**Don't build it until you actually need it.**

The temptation to build for hypothetical future requirements is strong. "What if we need to support ten million users?" You have two hundred. YAGNI says: build for what you know now, and trust that you can adapt later. Speculative features add complexity, maintenance burden, and decision fatigue — all for scenarios that usually never materialize. The best code you'll ever write is the code you didn't write.
